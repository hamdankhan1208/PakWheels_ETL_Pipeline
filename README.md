# PakWheels Real-Time ETL PipelineA comprehensive data engineering project that scrapes real-time car listings from PakWheels, processes the data through a Kafka-Spark streaming pipeline, and visualizes the results on a web dashboard.## Project Structure- `producer/`: Kafka producer for scraping and sending raw listing data.- `spark/`: PySpark streaming application for real-time data processing.- `dags/`: Airflow DAGs for orchestrating the ETL workflows.- `dashboard/`: A web-based application (app.py) to visualize processed car listing data.- `utils/`: Data cleaning and preprocessing utilities.- `sql/`: Database initialization scripts for storage.## Tech Stack- **Languages:** Python (PySpark, Flask)- **Streaming:** Apache Kafka- **Orchestration:** Apache Airflow- **Containerization:** Docker & Docker Compose- **Data Storage:** SQL (PostgreSQL/MySQL) and Parquet- **Web Scraping:** Selenium (using ChromeDriver)## Getting Started### Prerequisites- Docker and Docker Compose installed.- Python 3.8+ for local development.### Installation & Setup1. **Clone the repository:**   ```bash   git clone <repository-url>   cd <repository-folder>Launch Services:Use Docker Compose to start Kafka, Zookeeper, and the SQL database:Bashdocker-compose up -dInstall Dependencies:Bashpip install -r requirements.txtRun the Pipeline:Start the Kafka Producer: python producer/kafka_producer.pyStart the Spark Streamer: python spark/spark_streamer.pyLaunch the Dashboard: python dashboard/app.pyFeaturesReal-time Ingestion: Captures car listings (title, price, mileage, etc.) as they are published.Automated Cleaning: Standardizes engine capacities and price formats.Scalable Processing: Utilizes Spark structured streaming for high-throughput data handling.
